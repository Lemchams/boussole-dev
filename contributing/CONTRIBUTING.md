# Guide de contribution

Merci de vouloir contribuer à Boussole !

## 🛠️ Types de contributions

- **Corriger une erreur** → Issue avec label `bug`
- **Améliorer une ressource** → Pull Request
- **Améliorer un nœud** → Modifier le fichier Markdown dans `nodes/`
- **Proposer un nouveau domaine** → Issue avec label `new-domain`

## 📋 Standards de qualité

### Ressources
- Lien valide, auteur crédible
- Niveau clairement indiqué (debutant / intermediaire / avance)
- Indiquer si gratuit ou payant

### Nœuds Markdown
- Compétences formulées comme des actions vérifiables
- Checklist de validation à la fin
- Au moins 2 ressources et 1 projet par nœud

## 🔁 Processus de Pull Request

1. Fork le repo
2. Crée une branche : `git checkout -b feat/finance-add-options-trading`
3. Fais tes modifications
4. Ouvre une PR avec une description claire
5. Un mainteneur reviewera dans les 7 jours

## 🌍 Langues

Roadmaps rédigées en **français** par défaut. Traductions anglaises bienvenues dans `en/`.
